# 軌島 — Claude Design 打磨簡報（第五輪 · v0718i 基準 · 手術式範圍）

> 給 Claude Design 的工作說明。上傳 `design-mock.html` 這一個檔案即可開工。
> 第四輪（琺瑯 2.0）已完成整站 token 化重做；**本輪不是整站打磨**，
> 是四個明確議題的手術式修改＋一個選配題。與議題無關的區域請不要動。

## 這是什麼網站

台灣鐵道即時動態地圖「軌島」（https://railisland.tw）。台鐵／高鐵／各捷運依真實
時刻表＋即時誤點校正在地圖上跑，可點車跟隨、點站看班次、收集完乘章。
單檔 vanilla JS（index.html ~9000 行），地圖 Leaflet，列車與軌道畫在 canvas。

## 工作方式（重要）

- **只改 `design-mock.html`**。它的 `<style>` 與正式版 index.html 完全同步（自動抽取、
  逐 byte 一致），body 是各元件真實狀態的靜態複製——所見即正式版樣式。
- 改完交回整個檔案（或匯出 bundle），Claude Code 會 diff 出淨變更合併回 index.html。
- 檔尾 `<style id="mockOverrides">` 是 mock 專用展示樣式，**不要把設計寫進這塊**，合併時整段忽略。
- specimen 區（`.mock-spec`）＝canvas 繪製規格與色票；本輪只有議題 4 的徽章會碰到它。
- **亮暗雙主題都要給值**：`:root` 是亮（琺瑯）、`html[data-theme=dark]` 是暗（夜行）整組覆寫。
  預覽時手動把 `data-theme="dark"` 掛上 html 檢查暗色。

## 不可改的硬約束

1. **所有元素 id 不可改名**（JS 綁定）。
2. **JS 切換的 class／屬性不可改名**（樣式可改）：mock 內既有者為準，特別點名
   `active / on / off / run / show / cur / fs / sat / ambient / tools-open / sheet-open /
   fp-min / cexp / cfaint / playing / got / na / miss / past` 與 `html[data-theme]`。
3. **資料定義色不可改**（設計要能容納）：車種色——其他 `#8E44AD`、區間快 `#16A085`、
   區間車 `#2E6FB0`、自強 `#C0392B`、莒光/復興 `#E8792B`、高鐵 `#E85D0D`；
   路線／捷運官方色由資料檔經 inline style 或 canvas 進畫面。
4. DOM 巢狀盡量不動；「更多」抽屜的列是真 `<button class="ms-row">`（a11y 修正後），
   **不可改回 div**；抽屜列是正式工具鈕的 click 代理，結構別大改。
5. **捲軸鐵則**：工具帶的 6px 常駐細捲軸是自訂 `::-webkit-scrollbar`；
   **不可新增標準 `scrollbar-width` / `scrollbar-color`**（Blink 一設標準屬性，
   自訂捲軸整條隱形，這是踩過的坑）。
6. `?live=1` 是 OBS 訊號源契約：live 殼的版面不在本輪範圍，別動。

## 設計語彙（第四輪定調，可微調不可翻盤）

日式琺瑯站名牌×夜行地圖。米色紙感、藏青、印章紅、警戒金；硬投影 `0 2px 0`（非 blur）、
圓角 6–12px；CJK 字體棧 `-apple-system, "PingFang TC", "Noto Sans TC", sans-serif`、
數字 tabular-nums。所有顏色走 token（`--navy / --red / --gold / --ok / --warn-*` 家族）。

## 本輪議題（只做這些）

### 議題 1｜641–1360px 工具帶溢出（本輪最大題）

實測事實：桌面工具帶 `.controls` 內容總寬 **~1326px**；斷點 **641px** 即切桌面帶，
所以 iPad 直式 768px 只有 ~708px 可用（「站介紹」起七顆鈕在畫面外）、
**連 1280px 桌面都差 ~106px**（音樂控制被裁）。現行唯一提示是 6px 細捲軸，
已被兩位獨立 reviewer 判定不足。塞不下是物理事實——**別用縮字硬塞**。

主案（請以此出稿）：**(a) 手機殼上限 640 → ~900px**——iPad 直式改拿底部
tabbar＋「更多」抽屜（現成手機殼放大），桌面帶只服務 >900px；
並把 1280px 帶內的音樂控制收成圖示鈕＋彈出面板，讓帶內容寬壓進 ~1220px。
附念（可另出概念圖不實作）：(b) 桌面帶常駐「⋯ 更多」鈕，寬度不足時低優先鈕
自動收納、永不橫捲。底線：(c) 至少右緣漸層淡出＋捲軸提示強化。

注意：`fs`（全畫面）×面板×tabbar 的讓位組合是歷史盲區，主案改動斷點時
記得檢查 641–900 區間的 fs 版面。

### 議題 2｜手機群組頁籤捲動提示

實測：375px 下 `.grouptabs` 可視 195px／內容 369px，「北北桃」切一半、
「中南部」「縣市」完全在畫面外。要右緣漸層淡出（或等效提示）；
半截籤的自然暗示保留。同樣受硬約束 5（勿用標準 scrollbar 屬性）。

### 議題 3｜「更多」抽屜分組

現況：12 列可見＋1 個既有「動作」標題；前七列無標題、混雜。請重分組＋補段標題，建議：

- **地圖顯示**：外觀（亮/暗/自動）、衛星影像、軌道與路線、平交道記號、特殊站介紹、附近火車
- **資訊**：今日台鐵動態
- **觀看模式**：省電模式、背景音樂、放空模式
- **分享與資料**：分享畫面、Google 清單匯入（條件顯示）

預留回歸位（現被旗標隱藏、日後會回來）：錄影→觀看模式、帳號同步→分享與資料。
「定位我」死列已在 v0718i 移除，不用畫。段標題樣式沿用既有 `.ms-sec` 可微調。

### 議題 4｜「時刻表模擬／深夜重播」徽章脫紅

現況：這兩枚狀態徽章用 `--red`（#d23c2a）——但同一紅色在站內語意是**真異常**
（今日看板誤點≥5分徽章、平交道延誤），琥珀 `--warn` 是誤點標籤／公告、
綠 `--ok` 是 LIVE／準點。模擬徽章撞到警報色。

要求：模擬／深夜重播改為**非警報處理**（方向：灰藍、藏青 outline、或低彩度底），
仍要一眼可辨「這不是即時資料」；兩枚可同款或分款；亮暗主題都給值。
徽章底色由 JS inline 寫入——請在 specimen 區給出規格（class＋色值），合併時我們改 JS。

### 議題 5（選配）｜頁首副標字級

`.plate-sub`（ALL LINES · TAIWAN，隨頁籤動態換字）目前 8px 接近裝飾線。
若順手：放大到 10–11px 可讀，動態文案機制保留；`.plate` 的 grid 佈局
（`1fr|島|站名|1fr`）與 `.plate-foot` 鄰站帶**不可動**——logo 系統另有專輪處理。
