## A 組：參考 chienwen 網站的功能

背景結論（已驗證）：該站**僅台鐵**、走真即時資料（TDX 藏在自架後端 `railway-api.chienwen.net`），前端每 60 秒硬跳位、**無平滑動畫**。我們贏在動畫體驗＋四系統，它贏在即時性。值得抄的：

### A1. URL hash 深連結（優先做，成本最低價值最高）
- 格式參考它的：`#train:<車次>:follow`、`#station:<id>`。
- 行為：載入時解析 hash 直接進入跟隨/車站視圖；開始跟隨時寫回 hash（`history.replaceState` 避免污染上一頁）。
- 注意：我們是模擬時鐘，深連結是否也要帶模擬時間（如 `#train:123:follow&t=18:30`）——建議帶，否則分享出去對方看到的車位置不同。
- 驗收：複製連結開新分頁 → 直接在跟隨同一班車。

### A2. 「模擬非即時」誠實標示
- 它把推估值用斜體＋灰色誠實標記；我們全部是時刻表推估，應在 UI 明確標「依時刻表模擬，非即時位置」，避免誤解。一行標語即可，位置建議 lead 文案或時鐘旁。

### A3. 列車篩選
- 它有「全部／對號／區間／晚點」。我們版本：按車種（自強/區間…）或按路線篩選，畫面車太多時用。優先度低於 A1、A2。

### A4.（長期）若升級接 TDX 即時資料的架構範本
- TDX 金鑰**不進前端**，走自架後端代理（它前端 bundle 完全 grep 不到 TDX）。
- 輪詢 60 秒；失敗指數退避（10s 起跳、上限 600s）；`visibilitychange` 分頁回前景立即重拉。
- TDX 2024 起收費，quota 要省著用。此項不在本輪範圍，僅留架構筆記。

## B 組：放鬆掛機模式（ambient mode）

定位：像 lofi girl / flightradar24 掛機那類「開著陪伴」的內容——音樂＋自動換車跟隨，在螢幕上自己跑。

### 最小版本（第一輪做完就能上線）
1. **進入按鈕**「▶ 開始放空」：一鍵同時啟動音樂＋隱藏 UI＋啟動自動導演。這顆按鈕同時解掉瀏覽器 autoplay 限制（音訊必須由使用者互動觸發，無法繞過）。
2. **自動導演**：每 60–90 秒換一輛在跑的車。轉場節奏是體驗核心：先 `flyTo` 拉高俯瞰全島幾秒（看見整個台灣在跑）→ 緩降到下一輛車。挑車偏好長途車（自強/莒光比區間車有旅途感），偶爾跨系統切換（台鐵跟完切北捷）。
3. **音樂**：CC0／免版稅 lo-fi 音檔入 repo 循環播。注意檔案大小（幾 MB 內）。
4. **idle 隱藏 UI**：無操作數秒後淡出全部控制列，只留跟隨小牌＋時鐘；滑鼠動了淡回。
5. **Screen Wake Lock**：`navigator.wakeLock.request('screen')` 防螢幕休眠（需 https 或 localhost）。

### 第二輪（加分項）
6. **到站音效**（最有記憶點）：跟隨中的列車進站時播北捷進站音樂／台鐵月台鐘聲。音源版權要查清楚再放。
7. **深夜策略**：凌晨 00–05 台鐵幾乎沒車、捷運收班 → 偵測到深夜自動改播「18:00 尖峰重播」（模擬時鐘本來就支援任意時間，零成本），角落標「現在是深夜，重播 18:00 的台灣」。
8. **環境音**：鐵軌節奏聲、遠處平交道聲，比音樂耐聽，可與音樂混層。
9. **長時掛機穩定性**：marker／DOM 不得隨時間累積；實測掛過午夜（跨日換班表）不掛掉、記憶體平穩。
